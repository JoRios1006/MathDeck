I went through the CSV, and there are several systematic problems. The good news is they're not random math mistakes. They're the kind of bugs that come from the generator logic, which means fixing the code will fix hundreds of cards at once. Computers are wonderfully consistent that way. They make the same mistake 500 times without complaint.

## Overall verdict

I'd currently rate it:

* **CSV format:** 10/10 ✅
* **Pretty-printing:** 8/10
* **Mathematical correctness:** 4/10
* **Study usefulness:** 3/10

The main issue is that the generator is producing extremely degenerate polynomials.

---

# 1. Too many duplicates ❌

For example:

```
f(x)=x⁴
```

appears multiple times.

Likewise

```
f(x)=-2x²
```

```
f(x)=3x⁴
```

etc.

Every generated polynomial should be unique.

I'd recommend storing them in a `Set` before exporting.

---

# 2. The zero polynomial should NEVER be generated ❌

You have several cards like

```
f(x)=0
```

This creates nonsense study material.

Its root set is

```
All real numbers
```

Your program approximates this as

```
-20
...
20
```

which is mathematically incorrect.

The root set is **ℝ**, not "every integer between -20 and 20."

I would simply reject

```haskell
Polynomial [0]
```

during generation.

---

# 3. Integrals are wrong ❌

Example:

```
f(x)=x⁴

Integral

0/5x⁵ + C
```

should be

[
\frac{1}{5}x^5+C
]

Likewise

```
3x⁴

↓

0/5x⁵
```

should be

[
\frac35x^5
]

Your rational-number simplifier is clearly broken.

It's probably doing integer division.

Example:

```haskell
1 `div` 5
```

=

```
0
```

instead of

```
1/5
```

This is probably the single biggest bug.

---

Likewise

# Biggest design issue

The generator is creating **random coefficient lists**.

I think that's backwards.

Generate **roots**.

Example

```
[-2,1,5]
```

↓

Construct

[
(x+2)(x-1)(x-5)
]

↓

Expand.

Now you automatically get

* interesting coefficients
* integer roots
* good derivatives
* meaningful sign charts

No degenerate polynomials.

## Priority list

If this were my project, I'd fix things in this order:

1. **Fix the integral bug** (currently the most serious mathematical error).
2. **Never generate the zero polynomial.**
3. **Remove duplicate "Roots" sections.**
4. **Generate functions from random roots instead of random coefficients.** This will dramatically improve the quality of the deck.
5. **Deduplicate generated polynomials.**
6. **Add end behavior.**
7. **Add root multiplicities.**
8. **Expand to cubics and quartics with genuinely interesting structure.**

Once those are done, I think the quality of the deck jumps from "proof of concept" to something I'd actually use for daily Anki reviews. The mathematical engine is the heart of the project, and getting the generation strategy right will make every downstream feature better.
